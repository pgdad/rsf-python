@dataclass
class PrerequisiteCheck:
    name: str
    status: Literal["pass", "warn", "fail"]
    message: str

@dataclass
class ProviderContext:
    metadata: Any  # WorkflowMetadata
    output_dir: Path
    stage: str | None
    workflow_path: Path
    definition: Any = None
    auto_approve: bool = False
    stage_var_file: Path | None = None

class InfrastructureProvider(ABC):
    @abstractmethod
    def generate(self, ctx: ProviderContext) -> None: ...
    @abstractmethod
    def deploy(self, ctx: ProviderContext) -> None: ...
    @abstractmethod
    def teardown(self, ctx: ProviderContext) -> None: ...
    @abstractmethod
    def check_prerequisites(self, ctx: ProviderContext) -> list[PrerequisiteCheck]: ...
    @abstractmethod
    def validate_config(self, ctx: ProviderContext) -> None: ...

    def run_provider_command(
        self, cmd: list[str], cwd: Path | None = None, env: dict[str, str] | None = None,
    ) -> subprocess.CompletedProcess[str]:
        """Shared subprocess runner with capture_output=True."""

From src/rsf/providers/terraform.py (reference implementation):
```python
class TerraformProvider(InfrastructureProvider):
    def generate(self, ctx: ProviderContext) -> None:
        config = self._build_config(ctx)
        generate_terraform(config=config, output_dir=ctx.output_dir)

    def deploy(self, ctx: ProviderContext) -> None:
        self.run_provider_command(["terraform", "init"], cwd=ctx.output_dir)
        apply_cmd = ["terraform", "apply"]
        if ctx.stage_var_file is not None:
            apply_cmd.extend(["-var-file", str(ctx.stage_var_file)])
        if ctx.auto_approve:
            apply_cmd.append("-auto-approve")
        self.run_provider_command(apply_cmd, cwd=ctx.output_dir)

    def check_prerequisites(self, ctx: ProviderContext) -> list[PrerequisiteCheck]:
        if shutil.which("terraform"):
            return [PrerequisiteCheck("terraform", "pass", "terraform binary found")]
        return [PrerequisiteCheck("terraform", "warn", "terraform binary not found...")]
```

  <name>CDKProvider class with deploy, teardown, prerequisites, and bootstrap detection</name>

  <behavior>
    CDKProvider class (src/rsf/providers/cdk.py):
    - Inherits from InfrastructureProvider
    - Class attribute CDK_CMD = ["npx", "aws-cdk@latest"] -- all CDK operations go through npx
    - generate(ctx): builds CDKConfig from ctx.metadata (like TerraformProvider._build_config), calls generate_cdk(config, ctx.output_dir) from rsf.cdk.generator
    - deploy(ctx):
      1. First calls _check_bootstrap_or_warn(ctx) which uses boto3 to check for CDKToolkit stack
      2. If bootstrap missing, raises SystemExit with clear message showing `cdk bootstrap aws://ACCOUNT/REGION` command
      3. Builds command: CDK_CMD + ["deploy", "--app", "python3 app.py", "--output", "cdk.out"]
      4. If ctx.auto_approve: extends with ["--require-approval", "never"]
      5. If ctx.stage: extends with ["-c", f"stage={ctx.stage}"]
      6. Calls run_provider_command_streaming(cmd, cwd=ctx.output_dir) -- NOT run_provider_command (needs streaming)
    - run_provider_command_streaming(cmd, cwd, env): new method that runs subprocess.run WITHOUT capture_output, so stdout/stderr stream to terminal in real-time. Returns CompletedProcess. Uses shell=False, check=True, text=True.
    - teardown(ctx):
      1. Builds command: CDK_CMD + ["destroy", "--app", "python3 app.py", "--force"]
      2. If ctx.stage: extends with ["-c", f"stage={ctx.stage}"]
      3. Calls run_provider_command_streaming(cmd, cwd=ctx.output_dir)
    - check_prerequisites(ctx): returns list[PrerequisiteCheck]:
      1. Check shutil.which("npx") -- FAIL if missing with "npx not found. Install Node.js from https://nodejs.org/"
      2. Check shutil.which("cdk") -- WARN if missing with "cdk binary not found. Install with: npm install -g aws-cdk"
      3. Returns list of both checks
    - validate_config(ctx): no-op (same as TerraformProvider -- Pydantic handles config validation)
    - _build_config(ctx) -> CDKConfig: maps WorkflowMetadata fields to CDKConfig fields (same pattern as TerraformProvider._build_config)
    - _check_bootstrap_or_warn(ctx): uses boto3.client("cloudformation").describe_stacks(StackName="CDKToolkit"). On ClientError with "does not exist", raises SystemExit with message including the exact bootstrap command. On success, returns quietly. On unexpected errors, re-raises.
    - _get_account_and_region(): helper to get current AWS account ID and region for the bootstrap command message. Uses boto3.client("sts").get_caller_identity() for account and boto3.Session().region_name for region.

    Test cases:
    - test_cdk_provider_is_infrastructure_provider: CDKProvider() is instance of InfrastructureProvider
    - test_generate_delegates_to_generator: generate() calls generate_cdk with correct config (mock generator)
    - test_deploy_calls_streaming_subprocess: deploy() invokes npx aws-cdk@latest deploy (mock subprocess)
    - test_deploy_auto_approve_adds_flag: with auto_approve=True, command includes --require-approval never
    - test_deploy_stage_adds_context: with stage="prod", command includes -c stage=prod
    - test_deploy_checks_bootstrap_first: deploy() calls _check_bootstrap_or_warn before cdk deploy (mock boto3)
    - test_deploy_bootstrap_missing_raises: when CDKToolkit stack missing, raises SystemExit with bootstrap command
    - test_deploy_bootstrap_present_continues: when CDKToolkit exists, deploy proceeds normally
    - test_teardown_calls_destroy: teardown() invokes cdk destroy --force
    - test_teardown_stage_adds_context: with stage, command includes -c stage=X
    - test_check_prerequisites_npx_found: npx on PATH returns pass check
    - test_check_prerequisites_npx_missing: npx not on PATH returns fail check
    - test_check_prerequisites_cdk_found: cdk on PATH returns pass check
    - test_check_prerequisites_cdk_missing: cdk not on PATH returns warn check (not fail)
    - test_validate_config_noop: validate_config returns None without raising
    - test_streaming_subprocess_no_capture: run_provider_command_streaming does not use capture_output (check subprocess.run call args)
