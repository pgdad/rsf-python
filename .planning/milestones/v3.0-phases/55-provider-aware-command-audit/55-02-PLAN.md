# Plan 55-02: Diff command provider-aware graceful degradation

## Objective

Make `rsf diff` detect the configured provider before attempting Terraform state loading. When the provider is not Terraform, print a clear message that diff is not available for that provider, instead of crashing or showing misleading "No deployed state found" output.

## Must-Haves (goal-backward)

- [ ] `rsf diff` detects the configured provider before loading Terraform state
- [ ] When provider is not Terraform, prints a clear message explaining diff is not available
- [ ] Does not crash or show "No deployed state found -- showing all as new" for non-TF providers
- [ ] Exits with code 0 (not an error, just not applicable)
- [ ] Existing diff tests continue to pass
- [ ] New tests cover provider-aware graceful degradation

## Tasks

<task id="1">
### Task 1: Add provider detection and early exit to diff_cmd.py

**File:** `src/rsf/cli/diff_cmd.py`

**Actions:**

1. Add imports at the top of the file:
   ```python
   from rsf.config import resolve_infra_config
   ```

2. In the `diff()` function, after loading the local definition (after the `try/except` that calls `load_definition(workflow)`) and before the raw diff mode check, add provider detection:

   ```python
   # Detect configured provider
   try:
       infra_config = resolve_infra_config(local_def, workflow.parent)
       provider_name = infra_config.provider
   except Exception:
       provider_name = "terraform"

   # Diff only works with Terraform state -- gracefully decline for other providers
   if provider_name != "terraform":
       console.print(
           f"[yellow]Diff is not available for the {provider_name} provider.[/yellow]\n"
           f"The diff command compares local workflow definitions against Terraform state.\n"
           f"Your active provider ({provider_name}) does not use Terraform state."
       )
       raise typer.Exit(code=0)
   ```

3. This block must be placed AFTER `local_def = load_definition(workflow)` succeeds and BEFORE the `if raw:` block, so that provider detection happens before any Terraform state loading.

</task>
