# Plan 55-03: Watch command provider-based deploy routing

## Objective

Replace the hard-coded `terraform apply` in `rsf watch --deploy` with provider-based deploy routing. The watch loop (file monitoring, validate, generate) stays unchanged -- only the deploy step changes to go through the provider interface.

## Must-Haves (goal-backward)

- [ ] `rsf watch --deploy` routes deployment through the configured provider interface
- [ ] No hard-coded `terraform apply` command remains in watch_cmd.py
- [ ] The watch loop (file monitoring, validate, generate) is unchanged
- [ ] Deploy works for any configured provider (terraform, cdk, custom)
- [ ] Existing non-deploy watch tests continue to pass
- [ ] New tests verify provider-based deploy routing

## Tasks

<task id="1">
### Task 1: Replace hard-coded terraform deploy with provider routing in watch_cmd.py

**File:** `src/rsf/cli/watch_cmd.py`

**Actions:**

1. Add imports at the top:
   ```python
   from rsf.config import resolve_infra_config
   from rsf.providers import get_provider, ProviderNotFoundError
   from rsf.providers.base import ProviderContext
   from rsf.providers.metadata import create_metadata
   ```

2. Replace the entire deploy section in `run_cycle()` (the `if deploy:` block, approximately lines 82-117). The new implementation should:

   a. Resolve infrastructure config from the definition:
   ```python
   try:
       infra_config = resolve_infra_config(definition, workflow.parent)
   except Exception as exc:
       return False, f"{ts} Valid + regenerated, but deploy failed: could not resolve provider config: {exc}"
   ```

   b. Get the provider:
   ```python
   try:
       provider = get_provider(infra_config.provider)
   except ProviderNotFoundError as exc:
       return False, f"{ts} Valid + regenerated, but deploy failed: {exc}"
   ```

   c. Derive workflow name:
   ```python
   workflow_name = definition.comment or workflow.stem.replace("_", "-").replace(" ", "-")
   ```

   d. Create metadata and provider context:
   ```python
   metadata = create_metadata(definition, workflow_name, stage=stage)
   effective_output_dir = tf_dir or (workflow.parent / "terraform")
   if stage:
       effective_output_dir = effective_output_dir / stage
   ctx = ProviderContext(
       metadata=metadata,
       output_dir=effective_output_dir,
       stage=stage,
       workflow_path=workflow,
       definition=definition,
       auto_approve=True,
   )
   ```

   e. Run provider generate + deploy:
   ```python
   try:
       provider.generate(ctx)
       provider.deploy(ctx)
       return True, f"{ts} Valid + regenerated + deployed ({infra_config.provider})"
   except subprocess.CalledProcessError as exc:
       return False, f"{ts} Valid + regenerated, but deploy failed (exit {exc.returncode})"
   except Exception as exc:
       return False, f"{ts} Valid + regenerated, but deploy failed: {exc}"
   ```

3. Remove the now-unused `import shutil` from the top of the file (it was only used for `shutil.which("terraform")` in the old deploy block). Keep it if used elsewhere -- check first.

4. Rename the `tf_dir` parameter in `run_cycle()` to `output_dir` for provider-generic naming. Also update the `watch()` function's `tf_dir` option to add `output_dir` alias but keep backward compatibility:
   - Actually, keep the parameter name as `tf_dir` in `run_cycle` for backward compatibility with tests, but use it as a generic output directory internally. The variable inside the function can be `effective_output_dir`.

**Verification:**
