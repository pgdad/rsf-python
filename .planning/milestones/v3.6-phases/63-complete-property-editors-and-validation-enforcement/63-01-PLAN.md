    - Test: toggleExpand(id) when node has stateType "Task" and stateData.Resource is "arn:aws:..." should collapse normally (expandedNodeId becomes null)
    - Test: toggleExpand(id) when node has stateType "Succeed" (no required fields) should always allow collapse
    - Test: toggleExpand(id) on a collapsed node (expanding) should always work regardless of validation
    - Test: toggleExpand(id) for "Wait" with none of Seconds/Timestamp/SecondsPath/TimestampPath set should block collapse
    - Test: toggleExpand(id) for "Wait" with Seconds set should allow collapse
  </behavior>
  <action>
Define a REQUIRED_FIELDS map in flowStore.ts that maps stateType to an array of required field specs. Each spec is either a string (simple required field) or an object `{ oneOf: string[] }` for mutual-exclusion groups where at least one must be set.

Required fields per Pydantic model (fields with no default):
- Task: `Resource` is NOT actually a Pydantic field — Task inherits from _IOFields + _TransitionFields. Looking at the model, there is no `resource` field — it comes from the ASL spec but RSF's TaskState model does not have a `Resource` field as required. So Task has NO Pydantic-required fields (all have defaults or are handled by transition logic). Per CONTEXT.md: "Only Pydantic-required fields (no default in model) are marked required".
- Pass: No required fields (all optional)
- Wait: `{ oneOf: ['Seconds', 'Timestamp', 'SecondsPath', 'TimestampPath'] }` — validator enforces exactly one
- Succeed: No required fields
- Fail: No required fields (Error/Cause are optional)
- Choice: `Choices` is required (no default) but it's a complex array field shown read-only — skip validation since users can't edit it inline
- Parallel: `Branches` is required but complex/read-only — skip
- Map: No truly required scalar fields

So the only validation that matters for collapse-prevention is:
- Wait: must have at least one of Seconds/Timestamp/SecondsPath/TimestampPath

Update `toggleExpand` in the store:
1. If expanding (expandedNodeId !== nodeId), just set expandedNodeId = nodeId (no validation on expand)
2. If collapsing (expandedNodeId === nodeId), check REQUIRED_FIELDS for the node's stateType
3. For `oneOf` specs: check if at least one of the fields has a non-empty value in stateData
4. For simple string specs: check if that field has a non-empty value in stateData
5. If validation fails: set `toastMessage` to "Required field missing: {field description}" and do NOT set expandedNodeId to null
6. If validation passes: set expandedNodeId = null (collapse)

Also add a `collapseBlocked` transient state flag (string | null — set to nodeId when collapse is blocked, cleared on next toggleExpand call) so BaseNode can trigger the shake animation.
