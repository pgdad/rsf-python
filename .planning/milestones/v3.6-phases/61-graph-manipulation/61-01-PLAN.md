**1. Extend flowStore.ts:**
- Add to FlowState interface: `selectedEdgeId: string | null`, `toastMessage: string | null`
- Add actions: `selectEdge: (edgeId: string | null) => void`, `removeEdge: (edgeId: string) => void`, `clearSelection: () => void`, `setToastMessage: (msg: string | null) => void`
- `selectEdge` implementation: set `selectedEdgeId`, set `selectedNodeId = null` (mutual exclusion)
- Update `selectNode`: also set `selectedEdgeId = null` (mutual exclusion)
- `removeEdge` implementation: find edge by id, check edgeType. If 'catch' or 'choice', set toastMessage and return without removing. If 'normal' or 'default', filter it from edges array, clear selectedEdgeId.
- `clearSelection`: set both selectedNodeId and selectedEdgeId to null
- Initial values: `selectedEdgeId: null`, `toastMessage: null`

**2. Update TransitionEdge.tsx:**
- Import `useFlowStore` to read `selectedEdgeId`
- Compare `selectedEdgeId === id` to determine if this edge is selected
- When selected: override stroke color to `#3b82f6` (blue), set strokeWidth to 3, add `className="selected"` to the edge path
- When selected AND label exists: add blue highlight background (`#3b82f6` bg, white text) to the label div
- Keep all existing per-type color logic for non-selected state
- Set `interactionWidth={20}` on BaseEdge for easier click targeting

**3. Update GraphCanvas.tsx:**
- Import `selectEdge`, `removeEdge`, `clearSelection`, `selectedEdgeId`, `toastMessage`, `setToastMessage` from flowStore

- Update `handlePaneClick`: call `clearSelection()` instead of just `selectNode(null)`
- Add `useEffect` for keyboard listener on the graph container div (not document — prevents Monaco conflicts):

  - On Delete or Backspace: if `selectedEdgeId` is set, call `removeEdge(selectedEdgeId)` then `onGraphChange?.()`. If `selectedNodeId` is set, that will be handled in Plan 02.
  - On Escape: call `clearSelection()`
- Add `tabIndex={0}` to the graph-container div so it can receive keyboard focus
