  <action>
    **Create `deploy.sh`:**

    ```bash
    #!/usr/bin/env bash
    set -euo pipefail
    ```

    Structure:
    1. Compute `SCRIPT_DIR` via `$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)` — never rely on $PWD
    2. Set `TF_DIR="${SCRIPT_DIR}/terraform"`, `BUILD_DIR="${SCRIPT_DIR}/build"`, `METADATA_FILE="${RSF_METADATA_FILE}"`
    3. Parse command: `CMD="${1:-deploy}"`
    4. Extract metadata via jq:
       - `WORKFLOW_NAME=$(jq -r '.workflow_name' "${METADATA_FILE}")`
       - `EXECUTION_TIMEOUT=$(jq -r '.timeout_seconds // 86400' "${METADATA_FILE}")` — note `//` is jq's alternative operator for null fallback
    5. Case dispatch on `${CMD}`:
       - `deploy)`:
         - `mkdir -p "${BUILD_DIR}"`
         - `cd "${SCRIPT_DIR}"`
         - `zip -r "${BUILD_DIR}/function.zip" src/generated/ handlers/ -x "**/__pycache__/*" "**/*.pyc"` — zip RSF-generated source + handlers, exclude pycache
         - Print "Packaged function.zip"
         - `cd "${TF_DIR}"`
         - `terraform init -input=false`
         - `terraform apply -auto-approve -var="workflow_name=${WORKFLOW_NAME}" -var="execution_timeout=${EXECUTION_TIMEOUT}"`
         - Capture alias ARN: `ALIAS_ARN=$(terraform output -raw alias_arn)`
         - Print deploy complete message with sample `aws lambda invoke` command using the alias ARN
       - `destroy)`:
         - `cd "${TF_DIR}"`
         - `terraform init -input=false`
         - `terraform destroy -auto-approve -var="workflow_name=${WORKFLOW_NAME}" -var="execution_timeout=${EXECUTION_TIMEOUT}"`
         - Print "Teardown complete."
       - `*)`:
         - `echo "Unknown command: ${CMD}. Use 'deploy' or 'destroy'." >&2`
         - `exit 1`

    After creating deploy.sh:
    - `chmod +x examples/registry-modules-demo/deploy.sh`
    - `git update-index --chmod=+x examples/registry-modules-demo/deploy.sh` — so git tracks the execute bit

    **Create `rsf.toml`:**

    ```toml
    provider = "custom"

    [infrastructure.custom]
    program         = "/REPLACE/WITH/YOUR/ABSOLUTE/PATH/TO/examples/registry-modules-demo/deploy.sh"
    args            = ["deploy"]
    teardown_args   = ["destroy"]
    metadata_transport = "file"
    ```

    The placeholder path is deliberately obvious so users know to replace it. README.md (Task 2) will explain how.
