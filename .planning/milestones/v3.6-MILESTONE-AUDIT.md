### Phase 63: Complete Property Editors
- **End toggle UX:** End toggle in TaskNode/PassNode/ParallelNode/MapNode writes to `stateData` but `TRANSITION_MANAGED_KEYS` in `mergeGraphIntoYaml` silently discards it. The toggle visual changes but YAML `End` is controlled solely by edge topology. When a node has an outgoing edge, toggling "End: Yes" has no YAML effect. This is a deliberate architectural decision (tested at `mergeGraphIntoYaml.test.ts` line 297), but the UI doesn't communicate the constraint.
- **ChoiceNode Default UX:** Default text input in ChoiceNode writes to `stateData` but is silently discarded by `TRANSITION_MANAGED_KEYS`. Default in YAML is controlled exclusively by `default`-type edge connections. Input appears functional but has no YAML effect. Same deliberate design.

**Recommendation:** Disable End toggle when outgoing edge exists; make ChoiceNode Default input read-only showing connected edge target. These are UX polish items for a future milestone.
