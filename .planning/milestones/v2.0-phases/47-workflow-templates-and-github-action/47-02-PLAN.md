
**action/action.yml** — Composite GitHub Action definition:
```yaml
name: 'RSF Workflow Action'
description: 'Validate, generate, and optionally deploy RSF workflows. Posts plan summary as PR comment.'
branding:
  icon: 'check-circle'
  color: 'blue'

inputs:
  rsf-version:
    description: 'RSF version to install (e.g., "0.5.0"). Defaults to latest from PyPI.'
    required: false
    default: 'latest'
  rsf-install-source:
    description: 'Override RSF install source (e.g., git+https://... or local path). If set, rsf-version is ignored.'
    required: false
    default: ''
  workflow-file:
    description: 'Path to workflow.yaml file.'
    required: false
    default: 'workflow.yaml'
  deploy:
    description: 'Enable deployment after validate+generate. Set to "true" to enable.'
    required: false
    default: 'false'
  stage:
    description: 'Deployment stage (e.g., dev, staging, prod). Only used when deploy is true.'
    required: false
    default: ''
  auto-approve:
    description: 'Auto-approve Terraform changes during deploy. Set to "true" to skip confirmation.'
    required: false
    default: 'false'
  post-comment:
    description: 'Post plan summary as PR comment. Set to "false" to disable.'
    required: false
    default: 'true'
  python-version:
    description: 'Python version to use.'
    required: false
    default: '3.12'
  github-token:
    description: 'GitHub token for posting PR comments. Defaults to GITHUB_TOKEN.'
    required: false
    default: '${{ github.token }}'

outputs:
  validate-result:
    description: 'Validation result: pass or fail'
    value: ${{ steps.rsf-run.outputs.validate-result }}
  generate-result:
    description: 'Generation result: pass or fail'
    value: ${{ steps.rsf-run.outputs.generate-result }}

runs:
  using: 'composite'
  steps:
    - name: Set up Python
      uses: actions/setup-python@v5
      with:
        python-version: ${{ inputs.python-version }}

    - name: Run RSF
      id: rsf-run
      shell: bash
      env:
        RSF_VERSION: ${{ inputs.rsf-version }}
        RSF_INSTALL_SOURCE: ${{ inputs.rsf-install-source }}
        WORKFLOW_FILE: ${{ inputs.workflow-file }}
        DEPLOY: ${{ inputs.deploy }}
        STAGE: ${{ inputs.stage }}
        AUTO_APPROVE: ${{ inputs.auto-approve }}
      run: bash ${{ github.action_path }}/entrypoint.sh

    - name: Post PR comment
      if: inputs.post-comment == 'true' && github.event_name == 'pull_request'
      shell: bash
      env:
        GITHUB_TOKEN: ${{ inputs.github-token }}
        VALIDATE_OUTPUT: ${{ steps.rsf-run.outputs.validate-output }}
        GENERATE_OUTPUT: ${{ steps.rsf-run.outputs.generate-output }}
        DEPLOY_OUTPUT: ${{ steps.rsf-run.outputs.deploy-output }}
        VALIDATE_RESULT: ${{ steps.rsf-run.outputs.validate-result }}
        GENERATE_RESULT: ${{ steps.rsf-run.outputs.generate-result }}
        DEPLOY_RESULT: ${{ steps.rsf-run.outputs.deploy-result }}
        DEPLOY_ENABLED: ${{ inputs.deploy }}
        PR_NUMBER: ${{ github.event.pull_request.number }}
      run: python ${{ github.action_path }}/comment.py
```

**action/entrypoint.sh** — Shell script that:
1. Installs RSF based on `RSF_INSTALL_SOURCE` or `RSF_VERSION`:
   ```bash
   if [ -n "$RSF_INSTALL_SOURCE" ]; then
     pip install "$RSF_INSTALL_SOURCE"
   elif [ "$RSF_VERSION" = "latest" ]; then
     pip install rsf
   else
     pip install "rsf==$RSF_VERSION"
   fi
   ```
2. Runs `rsf validate $WORKFLOW_FILE`, captures output and exit code. Sets `validate-result` output.
3. If validate passes, runs `rsf generate $WORKFLOW_FILE`, captures output. Sets `generate-result` output.
4. If `$DEPLOY == "true"` and generate passes, runs `rsf deploy` with optional `--stage $STAGE` and `--auto-approve`. Captures output. Sets `deploy-result` output.
5. All outputs are set via `echo "validate-output<<EOF" >> $GITHUB_OUTPUT` pattern.
6. Script exits 0 even if validate fails (to allow comment posting). The validate-result output indicates pass/fail.
7. Make the script executable: `chmod +x action/entrypoint.sh`.

**action/comment.py** — Python script that posts a PR comment using the GitHub API:

1. Reads environment variables: `GITHUB_TOKEN`, `PR_NUMBER`, `GITHUB_REPOSITORY`, `VALIDATE_OUTPUT`, `GENERATE_OUTPUT`, `DEPLOY_OUTPUT`, `VALIDATE_RESULT`, `GENERATE_RESULT`, `DEPLOY_RESULT`, `DEPLOY_ENABLED`.

2. Formats a markdown comment body resembling a terraform plan:
   ```markdown
   ## RSF Workflow Plan

   | Step | Status |
   |------|--------|
   | Validate | :white_check_mark: Pass |
   | Generate | :white_check_mark: Pass |
   | Deploy   | :grey_question: Skipped |

   <details>
   <summary>Validation Output</summary>

   ```
   {validate_output}
   ```

   </details>

   <details>
   <summary>Generated Changes</summary>

   ```
   {generate_output}
   ```

   </details>

   ---
   *Posted by [RSF Action](https://github.com/owner/rsf-python)*
   ```

3. If validation failed, show a clear failure indicator with the error output expanded (not collapsed).

4. If deploy was enabled and ran, include deploy output in a collapsible section. If deploy included Terraform plan, show that too.

5. Uses `urllib.request` (stdlib only, no external dependencies) to POST the comment:
   ```python
   import json
   import os
   import urllib.request

   def post_comment(body: str) -> None:
       url = f"https://api.github.com/repos/{os.environ['GITHUB_REPOSITORY']}/issues/{os.environ['PR_NUMBER']}/comments"
       data = json.dumps({"body": body}).encode()
       req = urllib.request.Request(url, data=data, method="POST")
       req.add_header("Authorization", f"token {os.environ['GITHUB_TOKEN']}")
       req.add_header("Content-Type", "application/json")
       req.add_header("Accept", "application/vnd.github.v3+json")
       urllib.request.urlopen(req)
   ```

6. Before posting, check if a previous RSF Action comment exists on the PR (search for `## RSF Workflow Plan` marker). If found, update it instead of creating a new one (PATCH to `/comments/{id}`). This prevents comment spam on multiple pushes.
