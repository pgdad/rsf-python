  <behavior>
    CDK Engine (src/rsf/cdk/engine.py):
    - render_cdk_template("app.py.j2", stack_class_name="RsfStack", stack_id="rsf-my-workflow") produces valid Python with import and app.synth()
    - Uses standard Jinja2 delimiters ({{ }}, {% %}) since CDK templates produce Python, not HCL
    - Templates loaded from src/rsf/cdk/templates/ via FileSystemLoader

    CDK Generator (src/rsf/cdk/generator.py):
    - CDKConfig dataclass mirrors TerraformConfig but maps to CDK constructs: workflow_name, aws_region, triggers, dynamodb_tables, alarms, dlq_enabled, lambda_url_enabled, stage
    - CDKResult dataclass with generated_files and skipped_files lists
    - generate_cdk(config, output_dir) produces: app.py, stack.py, cdk.json, requirements.txt
    - GENERATED_MARKER = "# DO NOT EDIT - Generated by RSF" on first line of .py files; cdk.json and requirements.txt are always overwritten
    - _should_overwrite(path) returns True if file doesn't exist or first line starts with GENERATED_MARKER (reuse pattern from terraform/generator.py)
    - sanitize_stack_name(name) converts workflow_name to valid CDK stack ID (lowercase, hyphens, no special chars)
    - Stack class name = "RsfStack", stack ID = f"rsf-{sanitized_workflow_name}"

    Template: app.py.j2:
    - First line: # DO NOT EDIT - Generated by RSF
    - Shebang: #!/usr/bin/env python3
    - Imports aws_cdk as cdk
    - Imports stack class from stack module
    - Creates app, instantiates stack with stack_id, calls app.synth()

    Template: stack.py.j2:
    - First line: # DO NOT EDIT - Generated by RSF
    - Imports aws_cdk constructs (aws_lambda, aws_iam, aws_logs, etc.)
    - Class RsfStack(cdk.Stack) with __init__
    - Stage context: self.node.try_get_context("stage")
    - Lambda function using L2 aws_lambda.Function construct
    - IAM role with CloudWatch Logs, Lambda self-invoke, and durable execution permissions
    - CloudWatch log group with 14-day retention
    - Conditional blocks for: SQS triggers, DynamoDB tables, alarms, DLQ, Lambda URL (matching Terraform template coverage)

    Template: cdk_json.j2:
    - {"app": "python3 app.py", "watch": {"include": ["**"], "exclude": ["cdk.out"]}}

    Template: requirements.txt.j2:
    - aws-cdk-lib>=2.0.0
    - constructs>=10.0.0,<11.0.0

    Test cases:
    - test_render_cdk_template_basic: engine renders app.py.j2 with correct variable substitution
    - test_generate_cdk_creates_all_files: generator creates app.py, stack.py, cdk.json, requirements.txt
    - test_generate_cdk_generation_gap: existing user-edited file (no marker) is skipped
    - test_generate_cdk_overwrites_generated: existing file with marker is overwritten
    - test_generate_cdk_with_triggers: SQS trigger block appears in stack.py
    - test_generate_cdk_with_dynamodb: DynamoDB table block appears in stack.py
    - test_generate_cdk_with_dlq: DLQ block appears in stack.py
    - test_generate_cdk_with_lambda_url: Lambda URL block appears in stack.py
    - test_generate_cdk_minimal: minimal config produces valid files without optional resources
    - test_sanitize_stack_name: various inputs (PascalCase, kebab-case, special chars) produce valid IDs
  </behavior>
