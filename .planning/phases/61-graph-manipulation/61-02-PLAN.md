**1. Enhance removeState in flowStore.ts:**
- Before removing, check if `state.nodes.length <= 1`. If so, set `toastMessage = "Cannot delete the only state"` and return early (do NOT remove).
- If the node being removed has `isStart === true`, find the first remaining node sorted alphabetically by `id` (after removing current node) and set its `data.isStart = true`. The removed node's `isStart` is irrelevant since it's being deleted.
- Keep existing logic: filter out the node, filter out edges where source or target matches nodeId, clear selectedNodeId if it was the removed node.
- Also clear `selectedEdgeId` (in case user had an edge selected -- clean slate).

**2. Enhance mergeGraphIntoYaml.ts:**
- The existing code already handles most of this correctly:
  - States not in existingNodeIds are deleted from the AST (line 48-52)
  - Normal transitioning states with no outgoing normal edge get `End: true` (line 88-92)
  - StartAt is updated based on `isStart` flag on nodes (line 98-101)
- Add one piece of defensive logic: after updating transitions, verify that any state's `Next` value still points to an existing state. If `state.Next` is set but `!existingNodeIds.has(state.Next)`, delete `state.Next` and set `state.End = true`. This handles the case where a node deletion removes a target that was referenced. Apply same logic for `Default` on Choice states.
- This is a safety net -- the edge cascade in removeState should already remove the edges, but the AST-merge needs to be independently correct.

**3. Extend GraphCanvas.tsx keyboard handler (from Plan 01):**

  - This ensures Delete key works for both edges and nodes depending on what's selected
- Import `removeState` and `selectedNodeId` from flowStore (selectedNodeId may already be imported; removeState may need adding)
