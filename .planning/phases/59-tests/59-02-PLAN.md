**Module level:**
- Module docstring: `"""Integration tests for the registry-modules-demo example (TEST-02, TEST-03).\n\nDeploys via rsf deploy (custom provider pipeline), invokes a durable execution,\npolls to SUCCEEDED, verifies CloudWatch logs, and tears down via rsf deploy --teardown.\n"""`
- `pytestmark = pytest.mark.integration`
- Constants: `EXAMPLE_DIR = EXAMPLES_ROOT / "registry-modules-demo"`, `TF_DIR = EXAMPLE_DIR / "terraform"`
- `HAPPY_EVENT = {"image_id": "test-001", "format": "jpeg", "size_bytes": 1048576, "width": 1920, "height": 1080, "target_width": 960}`
- `RSF_TOML_PLACEHOLDER = "/REPLACE/WITH/YOUR/ABSOLUTE/PATH/TO/examples/registry-modules-demo/deploy.sh"`

**Helper function `_patch_rsf_toml()`:**
A context manager that:
1. Reads rsf.toml content
2. Replaces `RSF_TOML_PLACEHOLDER` with `str(EXAMPLE_DIR / "deploy.sh")`
3. Writes the patched content
4. Yields
5. In finally block, restores original content (even on failure)

**Class `TestRegistryModulesDemoIntegration`:**

**Fixture `deployment` (scope="class"):**
1. Use `_patch_rsf_toml()` context manager
2. Run `subprocess.run(["rsf", "generate", "workflow.yaml"], cwd=EXAMPLE_DIR, check=True, capture_output=True, text=True)` -- creates src/generated/
3. Run `subprocess.run(["rsf", "deploy", "workflow.yaml", "--auto-approve"], cwd=EXAMPLE_DIR, check=True, capture_output=True, text=True)` -- full custom provider pipeline
4. Read terraform outputs: `subprocess.run(["terraform", "output", "-json"], cwd=TF_DIR, check=True, capture_output=True, text=True)` -- parse JSON, extract alias_arn and function_name
5. Derive `log_group = f"/aws/lambda/{function_name}"` (no log_group_name terraform output exists -- Pitfall 5)
6. Call `iam_propagation_wait()`
7. Create exec_id via `make_execution_id("registry-modules-demo")`
8. Record `start_time = datetime.now(timezone.utc)`
9. Invoke Lambda: `lambda_client.invoke(FunctionName=alias_arn, InvocationType="Event", Payload=json.dumps(HAPPY_EVENT), DurableExecutionName=exec_id)` -- MUST use alias_arn not function_name (issue #45800)
10. Poll: `execution = poll_execution(lambda_client, alias_arn, exec_id)`
11. Yield dict: `{"execution": execution, "alias_arn": alias_arn, "function_name": function_name, "log_group": log_group, "start_time": start_time, "exec_id": exec_id}`
12. After yield (safety net cleanup): if terraform state is not empty, run direct `terraform destroy -auto-approve -input=false -var-file=terraform.tfvars.json` in TF_DIR as fallback. Then try to delete log group via `logs_client.delete_log_group(logGroupName=log_group)` catching ResourceNotFoundException.

**IMPORTANT fixture wrapping:** The rsf.toml must stay patched during the yield phase because the teardown test method needs it patched for `rsf deploy --teardown`. The fixture's post-yield cleanup restores the original.
