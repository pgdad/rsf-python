  <name>Task 1: Refactor deploy_cmd.py to use provider interface</name>
  <files>src/rsf/cli/deploy_cmd.py</files>
  <behavior>
    - deploy() with no infrastructure: block in YAML -> resolves to "terraform" provider -> identical output
    - deploy() with infrastructure: {provider: terraform} -> same as no config
    - deploy() --auto-approve -> provider receives auto_approve=True in ProviderContext
    - deploy() --stage prod -> provider receives stage and stage_var_file in ProviderContext
    - deploy() --code-only -> works for terraform, errors for non-terraform provider
    - deploy() --output-dir custom -> provider receives custom output_dir
    - deploy() --tf-dir custom -> still works (hidden alias for --output-dir)
    - deploy_cmd.py no longer imports TerraformConfig or generate_terraform directly
    - The _deploy_full() function's ~80-LOC extraction block is replaced by create_metadata()
  </behavior>
  <action>
    Rewrite deploy_cmd.py to follow this flow:

    1. Keep existing: file check, load definition, codegen generation, --no-infra early return
    2. Replace `--tf-dir` option with `--output-dir` (keep `--tf-dir` as hidden alias via typer)
    3. After codegen, resolve infrastructure config:
       ```python
       from rsf.config import resolve_infra_config
       infra_config = resolve_infra_config(definition, workflow.parent)
       ```
    4. Get provider:
       ```python
       from rsf.providers import get_provider, ProviderNotFoundError
       try:
           provider = get_provider(infra_config.provider)
       except ProviderNotFoundError as exc:
           console.print(f"[red]Error:[/red] {exc}")
           raise typer.Exit(code=1)
       ```
    5. Check --code-only constraint:
       ```python
       if code_only and infra_config.provider != "terraform":
           console.print("[red]Error:[/red] --code-only is only supported with the terraform provider")
           raise typer.Exit(code=1)
       ```
    6. Create metadata using create_metadata() from providers.metadata:
       ```python
       from rsf.providers.metadata import create_metadata
       metadata = create_metadata(definition, workflow_name, stage=stage)
       ```
    7. Build ProviderContext:
       ```python
       ctx = ProviderContext(
           metadata=metadata,
           output_dir=output_dir,  # was tf_dir
           stage=stage,
           workflow_path=workflow,
           definition=definition,
           auto_approve=auto_approve,
           stage_var_file=stage_var_file,
       )
       ```
    8. For code_only path: keep as Terraform-specific (direct subprocess call for targeted apply)
    9. For full deploy path: replace _deploy_full with:
       ```python
       provider.generate(ctx)
       # Check binary (provider can also do this, but keep explicit check for UX)
       provider.deploy(ctx)
       ```

    DELETE the entire _deploy_full() function and its ~80-LOC extraction block (lines ~114-195).
    The _deploy_code_only() function stays mostly the same but uses output_dir instead of tf_dir.

    IMPORTANT: Keep Rich console output messages for user feedback (generating, deploying, etc.).
    IMPORTANT: Handle stage_var_file derivation BEFORE creating ProviderContext (same logic as current: stages/{stage}.tfvars relative to workflow).
  </action>
  <verify>
    <automated>pytest tests/test_cli/test_deploy.py -x</automated>
  </verify>
  <done>deploy_cmd.py uses provider interface; no direct TerraformConfig/generate_terraform imports; ~80-LOC extraction block removed</done>
</task>
