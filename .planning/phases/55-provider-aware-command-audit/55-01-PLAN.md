# Plan 55-01: Doctor command provider-aware output

## Objective

Enhance `rsf doctor` output to visually distinguish which checks are relevant to the active provider, show the active provider name, and skip the `terraform/` directory check when Terraform is not the active provider.

## Must-Haves (goal-backward)

- [ ] `rsf doctor` with a CDK or custom provider configured shows Terraform binary check as WARN (not FAIL)
- [ ] Output visually distinguishes which checks are relevant to the active provider
- [ ] The `terraform/` directory check is skipped when Terraform is not the active provider
- [ ] Provider-specific prerequisite checks appear in the output
- [ ] JSON output includes `provider` field
- [ ] All existing doctor tests continue to pass
- [ ] New tests cover provider-aware output behavior

## Tasks

<task id="1">
### Task 1: Enhance doctor_cmd.py for provider-aware output

**File:** `src/rsf/cli/doctor_cmd.py`

**Actions:**

1. In `run_all_checks()`, make the `terraform/` directory check conditional on `provider_name == "terraform"`:
   - Change line ~339 from:
     ```python
     if tf_dir:
         results.append(_check_directory("terraform/", tf_dir))
     ```
   - To:
     ```python
     if tf_dir and provider_name == "terraform":
         results.append(_check_directory("terraform/", tf_dir))
     ```

2. In `_render_results()`, accept and display the active provider name:
   - Add `provider_name: str = "terraform"` parameter to `_render_results()`
   - Change the Environment header line from:
     ```python
     console.print("\n[bold]Environment[/bold]" if not no_color else "\nEnvironment")
     ```
   - To:
     ```python
     if provider_name != "terraform":
         label = f"\n[bold]Environment[/bold] [dim](provider: {provider_name})[/dim]" if not no_color else f"\nEnvironment (provider: {provider_name})"
     else:
         label = "\n[bold]Environment[/bold]" if not no_color else "\nEnvironment"
     console.print(label)
     ```

3. Update the `doctor()` function to pass `provider_name` to `_render_results()`:
   - In the `else` branch (non-JSON output), change:
     ```python
     _render_results(results, no_color=no_color)
     ```
   - To:
     ```python
     _render_results(results, no_color=no_color, provider_name=provider_name)
     ```

4. Add `provider` field to JSON output in `doctor()`:
   - In the `output_json` branch, add `"provider": provider_name` to the `report` dict, at the same level as `"checks"` and `"summary"`.

**Verification:**
- `python -m pytest tests/test_cli/test_doctor.py -x` passes
</task>
