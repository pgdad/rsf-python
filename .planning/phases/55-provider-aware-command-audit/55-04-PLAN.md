# Plan 55-04: Export command deduplication via create_metadata

## Objective

Replace the duplicated `_extract_infrastructure_from_definition()` function in `export_cmd.py` with the canonical `create_metadata()` from `providers/metadata.py`, converting the result via `dataclasses.asdict()`. Delete the duplicated function entirely.

## Must-Haves (goal-backward)

- [ ] `_extract_infrastructure_from_definition()` is removed from export_cmd.py
- [ ] `export_cmd.py` uses `create_metadata()` + `dataclasses.asdict()` instead
- [ ] `_build_sam_template()` works correctly with the new dict format
- [ ] SAM template output is identical (same keys, same values)
- [ ] All existing export tests pass with the new implementation
- [ ] Tests import from the correct location (no broken imports)

## Tasks

<task id="1">
### Task 1: Replace duplicate extraction with create_metadata in export_cmd.py

**File:** `src/rsf/cli/export_cmd.py`

**Actions:**

1. Add imports at the top:
   ```python
   from dataclasses import asdict
   from rsf.providers.metadata import create_metadata
   ```

2. Delete the entire `_extract_infrastructure_from_definition()` function (lines 26-115 approximately). This is the function that duplicates `create_metadata()` logic.

3. In the `export_workflow()` function, replace the call to the deleted function. Change:
   ```python
   infra = _extract_infrastructure_from_definition(definition, workflow_name)
   template = _build_sam_template(infra)
   ```
   To:
   ```python
   metadata = create_metadata(definition, workflow_name)
   infra = asdict(metadata)
   template = _build_sam_template(infra)
   ```

4. Verify key compatibility: The `asdict(WorkflowMetadata)` output has the same keys as the old function's dict output:
   - `workflow_name`, `handler_count`, `triggers`, `dynamodb_tables`, `alarms`, `dlq_enabled`, `dlq_max_receive_count`, `dlq_queue_name`, `lambda_url_enabled`, `lambda_url_auth_type`, `timeout_seconds`
   - Plus `stage` (which `_build_sam_template` doesn't use -- it will be ignored harmlessly)

5. Remove the now-unused import of `TaskState` from `rsf.dsl.models` (it was only used in the deleted function). Keep `StateMachineDefinition` since `_build_sam_template` type hint may reference it or other code uses it. Actually check: `StateMachineDefinition` is used in the deleted function's parameter type. After deletion, if no other function uses it, remove the import too.

**Verification:**
- `python -m pytest tests/test_cli/test_export.py -x` passes
</task>

### Task 2: Update export tests to use create_metadata

**File:** `tests/test_cli/test_export.py`

**Actions:**

1. Update imports: Replace:
   ```python
   from rsf.cli.export_cmd import (
       _build_sam_template,
       _extract_infrastructure_from_definition,
       _sanitize_logical_id,
   )
   ```
   With:
   ```python

   from rsf.cli.export_cmd import (
       _build_sam_template,
       _sanitize_logical_id,
   )
   from rsf.providers.metadata import create_metadata
   ```

2. Update all test methods in `TestBuildSamTemplate` that call `_extract_infrastructure_from_definition`:
   - Replace each occurrence of:
     ```python
     infra = _extract_infrastructure_from_definition(definition, "my-workflow")
     ```
   - With:
     ```python
     metadata = create_metadata(definition, "my-workflow")
     infra = asdict(metadata)
     ```
   - This affects tests 1 through 12 in `TestBuildSamTemplate`.

3. For test 12 (`test_extract_infrastructure`), rename it to `test_create_metadata` and update assertions:
   - The old test checks `infra["workflow_name"]` and `infra["handler_count"]` -- these still work with `asdict(metadata)`.
   - Also assert that `infra` has a `"stage"` key (new field from WorkflowMetadata, not in old function).

4. Verify all CLI command tests in `TestExportCommand` still pass without changes (they test end-to-end via CliRunner and don't import `_extract_infrastructure_from_definition`).

**Verification:**
- `python -m pytest tests/test_cli/test_export.py -x -v` all pass
</task>
