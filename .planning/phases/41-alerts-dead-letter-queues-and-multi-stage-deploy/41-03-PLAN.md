<objective>
Add multi-stage deployment support to the RSF CLI, allowing users to deploy to named stages (dev/staging/prod) with stage-specific variable overrides.

Purpose: Users need to maintain separate deployments for different environments (dev, staging, prod) without modifying core workflow definition files. The `--stage` flag controls which variable file is used and incorporates the stage name into resource naming to keep environments isolated.

Output: Updated deploy command with --stage option, stage-aware Terraform configuration, stage variable file resolution, and comprehensive TDD tests.
</objective>
