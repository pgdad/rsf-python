**## Step 9: Common Pitfalls (TUT-04)**

Per locked decision, dedicated section with each pitfall in Problem + Symptom + Fix structure. All 5 pitfalls:

1. **Absolute path in rsf.toml** — Problem: relative path or placeholder value. Symptom: "custom provider script not found: /REPLACE/..." error. Fix: echo "$(cd examples/registry-modules-demo && pwd)/deploy.sh" and paste into rsf.toml.

2. **deploy.sh not executable** — Problem: missing chmod +x. Symptom: "permission denied" error. Fix: chmod +x examples/registry-modules-demo/deploy.sh

3. **Packaging conflict** — Problem: running terraform apply directly instead of via deploy.sh. Symptom: "local_existing_package file does not exist: .../build/function.zip". Fix: always deploy via rsf deploy (deploy.sh creates the zip before calling terraform).

4. **Version pinning** — Problem: using ~> range constraints on registry module versions. Symptom: terraform init silently upgrades module, breaking tutorial instructions. Fix: use exact version strings (version = "8.7.0" not ~> 8.7). Why: Terraform locks provider versions in .terraform.lock.hcl but does NOT lock module versions.

5. **Durable IAM permissions** — Problem: Lambda deploys but durable executions fail. Symptom: "AccessDeniedException: ... lambda:CheckpointDurableExecution". Fix: hybrid approach — attach AWSLambdaBasicDurableExecutionRolePolicy managed policy PLUS inline supplement for InvokeFunction, ListDurableExecutionsByFunction, GetDurableExecution. Gotcha: attach_policies = true AND number_of_policies = 1 must BOTH be set.
