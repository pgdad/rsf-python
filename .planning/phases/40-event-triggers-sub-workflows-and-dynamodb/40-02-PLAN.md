<objective>
Add sub-workflow invocation support to the RSF DSL, enabling parent workflows to call child workflows as durable sub-executions.

Purpose: Users need to compose complex workflows from smaller, reusable child workflows. A Task state with a sub_workflow field invokes the named child workflow, and the generated orchestrator code handles the durable invocation.

Output: Extended TaskState model with sub_workflow field, semantic validation for sub-workflow references, and updated orchestrator template with sub-execution invocation code.
</objective>

  <action>
    **Design decision: How sub-workflows are referenced**

    Sub-workflows are declared at the top level and referenced from Task states by name. This follows the pattern of declaring resources (like triggers) at the top level and referencing them from states.

    1. In src/rsf/dsl/models.py, add a SubWorkflowRef model:
    ```python
    class SubWorkflowRef(BaseModel):
        """A reference to a child workflow that can be invoked as a sub-execution."""
        model_config = {"extra": "forbid"}
        name: str = Field(min_length=1)
        # The child workflow name — maps to a Lambda function name at runtime
    ```

    2. Add `sub_workflow` field to TaskState:
    ```python
    sub_workflow: str | None = Field(default=None, alias="SubWorkflow")
    ```
    This uses PascalCase alias because it's an ASL-style state field (like TimeoutSeconds, ResultPath). The value is the name of a declared sub-workflow.

    3. Add `sub_workflows` field to StateMachineDefinition:
    ```python
    sub_workflows: list[SubWorkflowRef] | None = Field(default=None, alias="sub_workflows")
    ```
    This uses snake_case alias because it's an RSF extension field (like rsf_version, lambda_url, triggers).

    4. In src/rsf/dsl/validator.py, add `_validate_sub_workflows(definition, errors)` called from `validate_definition()`:
       - If any TaskState references a SubWorkflow name that is NOT in the sub_workflows list, emit an error: "SubWorkflow '{name}' in state '{state_name}' is not declared in sub_workflows"
       - If a sub_workflow is declared but never referenced by any Task state, emit a warning
       - Walk states recursively (including Parallel branches and Map ItemProcessor) to find all SubWorkflow references

    DSL YAML example:
    ```yaml
    rsf_version: "1.0"
    sub_workflows:
      - name: payment-processor
      - name: inventory-checker
    StartAt: ValidateOrder
    States:
      ValidateOrder:
        Type: Task
        Next: ProcessPayment
      ProcessPayment:
        Type: Task
        SubWorkflow: payment-processor
        Next: CheckInventory
      CheckInventory:
        Type: Task
        SubWorkflow: inventory-checker
        End: true
    ```
  </action>

    2. In src/rsf/codegen/emitter.py, modify the Task state emitter to check `mapping.sub_workflow`:
       - If sub_workflow is set, emit code that invokes the child workflow via Lambda:
         ```python
         # Sub-workflow invocation: payment-processor
         response = lambda_client.invoke(
             FunctionName=f"{name_prefix}-{sub_workflow_name}",
             InvocationType="RequestResponse",
             Payload=json.dumps(current_data),
         )
         result = json.loads(response["Payload"].read())
         ```
       - If sub_workflow is NOT set, emit the normal handler call (existing behavior)

    3. In src/rsf/codegen/generator.py, pass sub_workflow information to the template context:
       - `has_sub_workflows`: bool — whether any state uses sub_workflows

       - Conditionally import boto3 and json when has_sub_workflows is True:
         ```python
         {% if has_sub_workflows %}
         import boto3
         import json
         lambda_client = boto3.client("lambda")
         {% endif %}
         ```
       - The state block emitter (emitter.py) handles the per-state logic; the template just needs the import.

    5. The child workflow function name at runtime follows the RSF naming convention: `{name_prefix}-{workflow_name}`. Since name_prefix is a Terraform variable (default "rsf"), the generated code reads it from an environment variable:
       ```python
       import os
       RSF_NAME_PREFIX = os.environ.get("RSF_NAME_PREFIX", "rsf")
       ```
       Then the invocation uses: `FunctionName=f"{RSF_NAME_PREFIX}-{sub_workflow_name}"`
