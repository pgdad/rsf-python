## Goal

Build a VS Code webview panel that renders a live, interactive graph preview of the current workflow.yaml file, updating as the user edits. The panel opens as a side split (like Markdown Preview), supports pan/zoom, click-to-navigate (clicking a node jumps to its YAML definition), and highlights states with validation errors using red borders.

## Must-Haves (Goal-Backward)

- [ ] Graph preview opens as a side panel (editor column split) matching Markdown Preview UX
- [ ] Graph renders all state types: Task, Pass, Choice, Wait, Succeed, Fail, Parallel, Map
- [ ] Graph updates live as user edits workflow.yaml (~500ms debounce, same as validation)
- [ ] Pan and zoom navigation in the graph panel
- [ ] Click-to-navigate: clicking a state node in the graph jumps the YAML editor cursor to that state's definition
- [ ] States with validation errors get red border/glow highlighting
- [ ] Graph layout uses a directed-graph algorithm for clean top-to-bottom flow
- [ ] Read-only preview — no editing in the graph
- [ ] Extension command registered: "RSF: Open Graph Preview" (available from command palette)

## Tasks

<task id="1" title="Build the graph preview webview provider and panel">
### What to build

1. **Update `vscode-extension/package.json`** (from plan 48-01) to add:
   - `contributes.commands`: `rsf.openGraphPreview` — "RSF: Open Graph Preview"
   - `contributes.menus`: Add command to editor title area for workflow.yaml files (icon: graph/preview)
   - `contributes.keybindings`: `Ctrl+Shift+V` for graph preview (matching Markdown Preview convention)

2. **src/graphPreviewProvider.ts** — Webview panel manager:
   - Implements `vscode.WebviewViewProvider` pattern
   - Creates a webview panel in the column beside the active editor
   - Manages panel lifecycle (create, update, dispose)
   - Listens for `textDocument/didChange` events from active workflow.yaml editor
   - Debounces updates (500ms) before sending new YAML content to webview
   - Sends messages to webview: `{ type: 'update', yaml: string, errors: Diagnostic[] }`
   - Receives messages from webview: `{ type: 'navigate', stateName: string }` — jumps editor cursor to state definition
   - Sets webview panel title to "RSF: {filename} Preview"
   - Handles panel visibility (pause updates when panel is hidden)

3. **src/graphPreview.ts** — Utility functions:
   - `yamlToGraph(yamlText: string)`: Parse workflow.yaml and convert to a node/edge graph structure
     - Each state becomes a node with: `id`, `label` (state name), `type` (Task/Choice/etc.), `position`
     - Transitions (Next, Default, Catch.Next, Choice branches) become directed edges
     - Parallel/Map states show branch indicators
   - `layoutGraph(nodes, edges)`: Apply dagre layout algorithm for clean top-to-bottom directed graph layout
     - Use `dagre` npm package (lightweight, no heavy dependencies)
     - Configure: `rankdir: 'TB'`, `nodesep: 50`, `ranksep: 80`
   - `applyErrorHighlights(nodes, errors)`: Mark nodes that have validation errors (for red border rendering)

4. **media/graphPreview.html** — Webview HTML:
   - Clean, minimal HTML shell that loads CSS and JS
   - Uses VS Code webview API for message passing
   - Includes nonce-based CSP for security

5. **media/graphPreview.css** — Graph styling:
   - Node styles by state type (different colors/shapes per type, matching the existing UI patterns):
     - Task: blue rounded rectangle
     - Choice: orange diamond-like shape (or hexagon)
     - Pass: gray rounded rectangle
     - Wait: purple rounded rectangle
     - Succeed: green rounded rectangle
     - Fail: red rounded rectangle
     - Parallel: blue dashed border
     - Map: blue dotted border
   - Edge styles: directed arrows with labels for Choice branch conditions
   - Error highlight: red border/glow on nodes with validation errors
   - Hover effect on nodes (pointer cursor, slight highlight)
   - Responsive layout fitting the panel width
   - Uses VS Code theme CSS variables for colors that adapt to light/dark themes

6. **media/graphPreview.js** — Client-side graph renderer:
   - Receives `update` messages with YAML content and errors
   - Renders nodes and edges using SVG (lightweight, no heavy framework needed)
   - Implements pan (mouse drag on background) and zoom (scroll wheel) with transform matrix
   - Click handler on nodes sends `navigate` message back to extension
   - Smooth transitions when graph updates (animate node position changes)
   - Handles empty/invalid YAML gracefully (show "Parse error" or "No states found" message)
