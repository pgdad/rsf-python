## Goal

Create the VS Code extension project with a Language Server Protocol (LSP) implementation that provides real-time YAML schema validation, semantic validation (matching `rsf validate` output), go-to-definition for state references, state name autocomplete, Find All References, and document highlights.

## Must-Haves (Goal-Backward)

- [ ] Extension activates only on files named `workflow.yaml` or `workflow.yml`
- [ ] Real-time validation with ~500ms debounce shows errors as inline squiggly underlines AND in Problems panel
- [ ] Schema-level validation uses bundled `rsf-workflow.json` (no external dependencies)
- [ ] Semantic validation checks: state reference resolution, reachability from StartAt, terminal state existence, States.ALL ordering — matching `src/rsf/dsl/validator.py` behavior
- [ ] Go-to-definition navigates from state names in Next/Default/Catch/Branches to state definitions
- [ ] Autocomplete dropdown for state names in Next/Default fields
- [ ] Find All References (Shift+F12) shows everywhere a state is referenced
- [ ] Document highlights: clicking a state name highlights its definition and all references
- [ ] Hover tooltips show error messages with quick-fix Code Actions for typo'd state names (Levenshtein-based suggestions)
- [ ] Status bar item shows validation status (checkmark with 0 errors or warning with error count)
- [ ] All validation and navigation works without local RSF installation

   - Activates the Language Server as a child process

   - Only processes files matching `**/workflow.yaml` or `**/workflow.yml`

<task id="2" title="Implement schema and semantic validation in TypeScript">
### What to build
Port the validation logic to TypeScript:

1. **src/validator.ts** — Schema-level validation:
   - Load bundled `rsf-workflow.json` schema
   - Use `ajv` with JSON Schema 2020-12 support to validate parsed YAML
   - Parse YAML using `yaml` library (which provides line/column source positions)
   - Map AJV validation errors to LSP `Diagnostic` objects with correct line/column ranges
   - Handle YAML parse errors gracefully (report syntax errors as diagnostics)

2. **src/semanticValidator.ts** — Semantic validation (port of `src/rsf/dsl/validator.py`):
   - **State reference resolution:** Check that all `Next`, `Default`, and `Catch[].Next` values reference existing state names in `States`
   - **Reachability check:** BFS from `StartAt` to verify all states are reachable (warn for unreachable)
   - **Terminal state check:** At least one state must be terminal (has `End: true`, or is `Succeed`/`Fail` type)
   - **States.ALL ordering:** In Retry/Catch arrays, `States.ALL` must be last
   - **StartAt validation:** `StartAt` must reference an existing state
   - **Recursive validation:** Validate Parallel branch definitions and Map ItemProcessor
   - **Timeout validation:** Warn if TimeoutSeconds > 2592000 (30 days)
   - Each error returns with YAML document position (line/column) for precise diagnostics
   - Match severity levels from Python validator (error vs warning)

3. **src/stateNameProvider.ts** — State name intelligence:
   - **Go-to-definition:** For state names in Next/Default/Catch/Branches fields, return the position of that state's definition in the States mapping
   - **Find All References:** Given a state name at cursor, find all places it is referenced (Next, Default, Catch.Next, StartAt, Branches)
   - **Document Highlights:** Same as references but within current document (highlight definition + all references)
   - **Autocomplete:** When cursor is in a Next/Default value position, offer all defined state names as completion items
   - **Hover:** Show state type and any validation errors for the referenced state
   - **Code Actions:** For unresolved state references, suggest closest matching state names using Levenshtein distance (e.g., "Did you mean 'ProcessOrders'?")

   Implementation approach:
   - Parse YAML to AST to get exact positions of all state definitions and state references
   - Build a map: `stateName -> { definitionRange, referenceRanges[] }`
   - Use `yaml` library's CST/AST for position-accurate node lookups
