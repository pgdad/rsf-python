Key understanding:
- RSF uses Jinja2 templates with custom delimiters (<< >> for variables, <% %> for blocks) in `src/rsf/terraform/templates/`
- Templates render to .tf files via `src/rsf/terraform/engine.py` and `src/rsf/terraform/generator.py`
- The generator passes a `context` dict with keys like `resource_id`, `workflow_name`, `name_prefix`, etc.
- `workflow_name` is already available in all template contexts (used in main.tf.j2 output_path)
- Examples have "# DO NOT EDIT - Generated by RSF" markers on generated files; hand-edited files (like dynamodb.tf in order-processing) do NOT have this marker
- The registry-modules-demo already has `ManagedBy = "rsf"` and `Workflow = var.workflow_name` tags on all modules — no changes needed there
- The CLI plugin templates (s3-event-pipeline, api-gateway-crud) already have `ManagedBy = "rsf"` tags — no changes needed there

Tag convention to follow (matching existing patterns in registry-modules-demo and CLI templates):
```hcl
tags = {
  ManagedBy = "rsf"
  Workflow  = var.workflow_name   # or << workflow_name >> in Jinja2 templates
}
```

AWS resources that support tags (relevant to this project):
- aws_lambda_function (main.tf.j2)
- aws_iam_role (iam.tf.j2) — supports tags
- aws_cloudwatch_log_group (cloudwatch.tf.j2) — supports tags
- aws_dynamodb_table (dynamodb.tf.j2) — supports tags
- aws_sns_topic (alarms.tf.j2) — supports tags
- aws_cloudwatch_metric_alarm (alarms.tf.j2) — supports tags
- aws_sqs_queue (triggers.tf.j2 for SQS triggers, dlq.tf.j2 for DLQ) — supports tags
- aws_lambda_function_url (lambda_url.tf.j2) — does NOT support tags (it is not a taggable resource)
- aws_cloudwatch_event_rule (triggers.tf.j2) — supports tags
- aws_lambda_permission — does NOT support tags
- aws_lambda_event_source_mapping — does NOT support tags
- aws_sns_topic_subscription — does NOT support tags
- aws_cloudwatch_event_target — does NOT support tags
- aws_iam_role_policy — does NOT support tags (inline policy)

Note: versions.tf.j2 does not exist yet as a template. versions.tf files in examples are hand-written per example. No changes needed there.
